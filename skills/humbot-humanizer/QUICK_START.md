# Humbot Humanizer - Quick Start Guide

Transform AI-generated text into natural, human-like writing.

## 🚀 Basic Usage

Just ask me to humanize your text! I'll handle the API calls automatically.

### Examples

**Example 1: Direct text**
```
You: "Humanize this: Artificial intelligence has revolutionized numerous 
industries by providing innovative solutions..."

Me: [calls API, shows you the humanized version with stats]
```

**Example 2: From a file**
```
You: "I have an essay in essay.txt that sounds too AI-ish. Can you humanize it?"

Me: [reads file, processes it, shows results]
```

**Example 3: Improve what I just wrote**
```
You: "Write me a blog post about healthy eating"
Me: [writes blog post]

You: "That sounds too robotic, humanize it"
Me: [runs it through Humbot, gives you a more natural version]
```

## 📊 What You Get

After processing, I'll show you:
- **Humanized text** - The natural-sounding version
- **Detection score** - How "human" it sounds (0-100, higher is better)
- **Word counts** - Before and after
- **Model used** - Quick/Standard/Advanced

Example output:
```
✨ HUMANIZED TEXT
The integration of machine learning algorithms allows organizations 
optimise its operations and improves productivity...

📈 STATS
Original: 65 words → Humanized: 56 words
Detection: HUMAN (score: 85/100)
Model: Quick
```

## ⚙️ Model Types

- **Quick** (default) - Fast results, good quality (5-10 seconds)
- **Standard** - Better quality (10-20 seconds)  
- **Advanced** - Best quality (20-60 seconds)

You can request a specific mode: "Humanize this using Advanced mode"

## 💡 Tips for Best Results

1. **Length matters**: Minimum 50 words required. Longer texts (100+) work better.
2. **Context helps**: More text gives the API better context for natural rewriting.
3. **Review output**: Always check the result - it may need minor tweaks.
4. **Try different modes**: If Quick isn't good enough, try Advanced.
5. **Multiple attempts**: Sometimes running it again produces different (better) results.

## ⚠️ Important Notes

### What This Does
- Makes AI-written text sound more natural
- Helps pass AI detection tools
- Improves flow and readability
- Removes robotic patterns

### What This Doesn't Do
- Grammar correction (use editing tools)
- Translation (different skill)
- Content generation (only rewrites existing text)
- Fact-checking (always verify important details)

## 🎯 When to Use

**Good use cases:**
- Academic papers that sound too formal
- Blog posts that feel robotic
- Essays generated by ChatGPT/Claude
- Marketing copy that needs a human touch
- Any text that "sounds like AI wrote it"

**Not ideal for:**
- Very short snippets (under 50 words)
- Already natural-sounding human writing
- Code or technical documentation
- Legal or medical content requiring exact wording

## 🔧 Manual Usage (Advanced)

If you want to run the scripts directly:

```bash
# Create a task
cd ~/.openclaw/workspace/skills/humbot-humanizer
python scripts/create_task.py "Your text here" --model-type Quick

# This outputs a task_id, then:
python scripts/retrieve_result.py <task_id> --wait

# Or in one line:
TASK_ID=$(python scripts/create_task.py "Your text" --model-type Quick 2>/dev/null | tail -1)
python scripts/retrieve_result.py $TASK_ID --wait
```
